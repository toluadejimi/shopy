What is Shopy?

  • Shopy is a Laravel, Flutter and Vue Framework based multi-vendor Food, Grocery, eCommerce & pharmacy online platform.

  • We provide Customer, Seller & Delivery boy Flutter app (Android & iOS) with great features

  • Admin & Seller panel has all the control of your shop and system, with clear and minimal design

Core Features

  • Multi Language + RTL: We provide multi-language supports with RTL layout for specific languages

  • Theme & Layout: We provide Light, Dark & Device theme mode. You can also customize the layout with the left menu size and a full layout to hide all menus.

  • Module: Shopy contains food, grocery, eCommerce & pharmacy module. So most of business done in those modules

  • Zone: Zone is better option to represent your shop in your area. So customers can't see you shop out of your zone

  • Shop Plan (Subscription): Seller can choose of of your best plan to start a shop. All plans have different advantages

  • Coupon: Offers can attract more customers as more revenues. Coupon have module, zone, shop based & other controls

  • Wallet: Customer can add money into wallet and use it when order anything

  • POS : Point of Sales is useful to take order at at shop or restaurant

  • Employee : If you need more than one person can control then create employee for your shop

  • Role Based : Role is restrict uses of your employee. Security is our upmost priority

  • Notification : We use FCM notification for orders status. Admin can also send a notification for latest updates or new offers

  • Loyalty Wallet : Every order give loyalty point to customer, that point convert to wallet money & purchase anything from wallet

  • Refer & Earn : When customer create an account it they can enter referral code. Then some money added to wallet.

  • Review System : Customer can provide a review to particular shop, product and delivery boys. So that other customer can see and take decisions.

  • Reports : Admin & Seller can see report for every day or any range of dates. It helps to keep track of revenues

  • Addons : There is some product that can have many addons. So seller can attach addons to that product

  • Units : Unit is useful for same product with multiple options, It's look like Pizza sizes are S, M, L

  • Dashboard : Admin and seller need to see everything at one place, It's dashboard. We provide beautiful with minimal design dashboard

  • Home Layout : Admin can control what see's on customer app home. So admin can advertise any shop and products

  • Invoice : When order is creating then customer will receive invoice over mobile number and email. Customer can also access from mobile app

  • Order OTP : OTP is send over mobile and apps. It need when delivery boy deliver any orders.

  • Print : You can print an invoice if you have attached printers or download pdf for order.

  • Newsletter : Customer can join your newsletter, Where admin can see all the subscribers and sent a promotional email to all

  • File Manager : Admin can see all images (Files) at a panel. So it will useful when editing or creating any products

  • Category : Category & sub category useful at search and filter out products.

  • Assign Delivery : When order is ready then automatically delivery boy assign that order or seller can also assign manually

  • & Much more : Their so many other features available. You need to take a look at admin panel.

Order Flow

  • Customer can place order from web or mobile app

  • If order need prescription then customer need to upload prescription (as image) at checkout

  • Seller can accept, cancel or reject based on product stock or any other circumstances

  • If order is rejected then customer resubmit same order with needful prescription

  • Seller or customer can cancel order before order is accepted. If any one cancel the order and payment was paid online then all payment revert to customer wallet

  • If seller is accept then product stock is deduct automatic

  • Then seller can ready order that time server automatic assign delivery boy, which is nearest

  • Or seller can change delivery boy and assign manually delivery boy

  • Then delivery boy can accept or reject order [If you enable this feature from admin panel]

  • If delivery boy accept then he/she need to go that shop and pickup order

  • Then delivery boy need to go customer home address and delivery order. That time delivery boy need otp to deliver [If you enable this feature from admin panel]
